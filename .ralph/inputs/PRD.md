## 1. 서비스 개요


### 1.1 서비스 비전


PlantFit은 공간의 일조량을 AI로 분석하여 최적의 식물을 추천하고, 식물을 의인화된 캐릭터로 케어할 수 있는 플랜테리어 도우미 서비스입니다.


### 1.2 핵심 가치


- **내 공간에 맞는 식물 추천** — 일조량 분석 기반 과학적 매칭
- **쉬운 케어** — 의인화 알림으로 식물과 감성적 유대감 형성
- **직관적 배치** — 2D 평면도로 플랜테리어 시뮬레이션


### 1.3 타겟 사용자


- 식물을 키우고 싶지만 어떤 식물을 어디 두어야 할지 모르는 초보자
- 플랜테리어에 관심 있는 홈 인테리어 관심층
- 반려 식물에 감성적 유대감을 원하는 MZ세대

## 2. Phase별 기능 요구사항 (MVP)


### PHASE 1 — 진입 & 온보딩


#### STEP 1. 앱 첫 실행


- 첫 진입 시 "내 공간에 맞는 식물 찾아줄게요" 카피와 함께 시작 화면 노출
- 별도 로그인 없이 바로 시작 가능
- 분석 데이터 없을 시 결과 영역에 분석 유도 카피 노출
- 공간 분석 완료 후 해당 페이지에서 결과 확인 가능


#### STEP 2. 공간 분석 방식 선택


- "우리 집 분석하기" 버튼 탭 시 분석 방법 2가지 선택지 제시
  - 📷 사진 업로드
  - 📍 주소 입력 (공공API 자동 분석)


---


### PHASE 2 — 공간 데이터 수집


#### STEP 3-A. 사진 업로드 플로우


- 창문이 보이는 방 사진 1~5장 업로드 **(확정: 최대 5장)**
- AI 가 사진에서 창문 위치·방향·크기 추정
- AI 추정 결과를 사용자에게 확인 질문 (예: "이 창문이 남향 맞나요?")
- 아니오 응답 시 → 8방위 버튼 선택 UI로 방향 직접 입력 (남 / 남동 / 동 / 북동 / 북 / 북서 / 서 / 남서)


#### STEP 3-B. 주소 입력 + 공공API + 평면도 크롤링 플로우


- 주소 텍스트 자유 입력 (대략적 입력 허용) → LLM이 정확한 주소 후보 리스트 제안 → 사용자 선택
- 사용자가 해당 건물의 동·호수를 추가 입력
- 건축물대장 공공API로 건물 층수·면적·건축연도 자동 조회
- 호갱노노 사이트에서 해당 주소·호의 **평면도 이미지 + 건물 방위 각도**를 Playwright로 자동 크롤링
- AI가 **평면도 이미지 + 건물 방위 각도 + 건축물대장 정보 + 건축법령상 규정**을 종합 분석하여 창문 위치·크기·방향을 도출




---


### PHASE 3 — 일조량 분석 & 결과 도출


#### STEP 4. 채광 분석 처리


- 수집된 데이터(사진 + 주소 + 건축 정보 + 평면도 이미지)를 AI가 종합 분석
- 태양 위치 기반 직달일사 시간 계산 
- 최종 채광 등급 산출 (강 / 중 / 약)
- 로딩 중 친근한 진행 문구 노출 (예: "햇빛 경로를 계산하고 있어요 🌞")


#### STEP 5. 공간 채광 결과 화면


- 방별 채광 등급 표시 (예: 거실 창가 ☀️ 강함 / 침실 구석 🌥 약함)
- 2D 평면도 위에 채광 구역 색상으로 표시 (3D 제외)


---


### PHASE 4 — 식물 추천


#### STEP 6. 식물 추천 리스트


- 채광 등급 기반 공간별 적합 식물 3~5개 추천
- 각 식물 카드: 이름, 난이도, 추천 이유, 예상 효과 (공기정화, 인테리어 포인트 등) 표시
- "키우기 쉬운 것만 보기" 초보자 필터 기본 ON
- 추천 외 식물 직접 검색 등록 지원 **(MVP 포함)**: 텍스트 검색 → DB 매칭 → 없으면 직접 등록 폼


#### STEP 7. 배치 추천


- 추천 식물을 어느 자리에 놓을지 2D 평면도 위에 아이콘으로 표시
- 추천 및 등록된 식물은 2D 평면도 위에 배치/삭제 가능
- 배치 시 텍스트 설명 병행 (예: "이 식물은 거실 창가 왼쪽이 딱 맞아요")
- 햇빛이 창문을 통해 집 안을 스캔하듯 들어오는 애니메이션 → 채광 최적 지점에서 멈춤
- 햇빛 良 구역 배치 시 긍정 컬러(그린), 음지 구역 배치 시 부정 컬러(레드) 표시
- 하단 또는 우측 영역에 2D 평면도에 놓인 식물 리스트 확인 및 관리


---


### PHASE 5 — 식물 등록 & 의인화 케어


#### STEP 8. 식물 선택 & 등록


- 추천 리스트에서 키울 식물 선택


#### STEP 9. 식물이 먼저 말 거는 케어 알림


- 식물마다 의인화하여 상태를 한 줄 코멘트로 송출
- 물 줄 시간 → "나 목말라… 🥺"
- 오래 방치 → 캐릭터 표정 시들어짐 + "요즘 나 잊은 거야?" 말풍선


- 물 안 준 경우 → "너 변했어" / "진짜 이러기야?" / "나한테 할 말 없어?"


#### STEP 10. 케어 기록 & 성장 트래킹


- 물주기 / 분갈이 / 영양제 기록 탭
- 식물 상태 변화 로그 (잘 자라고 있어요 / 주의 필요)
- 과거 성장 기록 확인 캘린더 버튼 (시연에서는 캘린더 전체화면 미노출)

## 4. 에러 & 예외 처리


| 상황 | 처리 방식 |
|------|-----------|
| 건축물대장 API 조회 실패 | "건물 정보를 불러오지 못했어요" 안내 후 수동 입력 폼 제공 |
| 호갱노노 평면도 크롤링 실패 | "평면도를 가져오지 못했어요" 안내 후 건축물대장 + 건축법령 기준으로만 분석 진행 |
| 사진에서 창문 인식 실패 | "창문을 찾지 못했어요" 안내 후 방향 직접 입력 유도 |
| 비아파트 건물 (단독주택 등) | 평면도 크롤링 불가 시 건축물대장 조회 결과 기반 최대한 분석, 부족 시 사용자에게 추가 정보 요청 |
| AI 사진 분석 응답 지연 / 실패 | 재시도 1회 후 실패 시 "분석에 실패했어요" 안내 및 재시작 유도 |
| 창문 크기 분석 실패 | 보정계수 1.0(기본값) 적용 후 분석 계속 진행, 결과 화면에 "창문 크기 미반영" 안내 |
| 식물 DB 검색 결과 없음 | "찾는 식물이 없어요. 직접 등록할까요?" → 직접 등록 폼 제공 |

## 5. 비기능 요건


### 8.1 성능 목표


| 항목 | 목표값 |
|------|--------|
| 채광 분석 전체 응답 시간 | 30초 이내 (로딩 UX 필수) |
| 건축물대장 API 응답 | 최대 3초 (초과 시 로딩 인디케이터 노출) |
| 사진 업로드 처리 | 장당 최대 5초 이내 |
| 페이지 초기 로딩 | 3초 이내 (LCP 기준) |


### 8.2 보안 / 개인정보


- 위치정보 수집 시 개인정보처리방침 및 위치정보 이용약관 고지 필수 (위치정보보호법 준수)
- 사진 업로드 데이터는 AI 분석 후 서버 미보관 (비로그인 서비스 특성)
- HTTPS 필수


### 8.3 접근성 (a11y)


- 주요 버튼 및 입력 필드 키보드 접근 가능
- 이미지 alt 텍스트 제공
- 색상만으로 상태를 구분하지 않음 (채광 등급은 색상 + 텍스트 병행)
